**Status:** Accepted
**Type:** implementation

## Outcome

Interpreter and IR match Temurin 25 on the three fixed dynamic Proxy fixtures
as an optional Java SE compatibility capability. Proxy defines and caches a
runtime Class by defining loader and ordered interfaces, associates each
instance with one InvocationHandler, dispatches interface/Object/default
methods through typed values, and preserves declared/undeclared exception
rules. AOT precisely rejects the capability.

This slice no longer closes all R3 work and does not depend on
StringConcatFactory or LambdaMetafactory.

## Scope

- Loader-aware ordered-interface Proxy Class cache over the generated-class
  kernel and generated typed methods.
- Bounded Proxy new/is/get-handler surface and fixed-row input validation.
- InvocationHandler dispatch with boxing/unboxing/return checks.
- `equals`/`hashCode`/`toString` Method facades and dispatch.
- `InvocationHandler.invokeDefault` with invokespecial-equivalent target.
- RuntimeException/Error/declared checked/undeclared checked exception rules.
- Proxy-specific concurrency stress and compatibility evidence.

## Non-scope

Catty Runtime generated-object APIs; non-public interface/module/package
combinations beyond fixed rows; serialization/RMI; broad custom loaders;
arbitrary default-method conflicts; arbitrary generated byte-array classes;
hidden classes; concat/lambda completion; full 24-row closure; or AOT
execution/fallback.

## Semantic constraints

This is an optional Java SE API contract. Cache key includes defining loader and
ordered interface identities; interface order is significant. One fully linked
runtime Class is published per key. Each instance holds one handler. Handler
and throwable identities are preserved. Object methods dispatch as
Object-declared Method facades. Default body execution occurs only through the
explicit default-invocation operation.

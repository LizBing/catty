# Work log

The work log records concise, factual project events that are too detailed for
`PROJECT_STATUS.md`: contracts opened or closed, significant test evidence,
integration decisions, and handoffs. It is not a transcript and should not
repeat implementation details already preserved in commits.

Use one file per month (`YYYY-MM.md`) and append dated entries. State the agent
or human actor, related workstream, commits, evidence, and next action.
